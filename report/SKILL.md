# report - 企业风险分析报告生成

整合新闻、财务、风险等多维度数据，生成综合企业风险分析报告，支持多种输出格式。

## 功能特性

### 核心能力
- **数据整合**：汇集新闻、风险、财务等多源数据
- **智能分析**：基于规则和模型的风险评估
- **报告生成**：结构化的风险分析报告
- **多格式导出**：支持 Text/Markdown/PDF/HTML/JSON
- **模板定制**：支持自定义报告模板
- **版本管理**：保留历史报告版本

### 分析维度
- 企业基本信息分析
- 新闻舆情分析
- 风险事件分析
- 财务健康度分析
- 行业对比分析
- 综合风险评估

## 使用方法

### 基础用法
```bash
./generate.sh "公司名称"
./generate.sh "华为技术有限公司"
./generate.sh "宁德时代"
```

### 高级用法
```bash
# 完整分析报告
./generate.sh "公司名称" --full

# 仅风险分析
./generate.sh "公司名称" --type risk

# 仅财务分析
./generate.sh "公司名称" --type finance

# 快速概览
./generate.sh "公司名称" --quick

# 指定输出格式
./generate.sh "公司名称" --format markdown
./generate.sh "公司名称" --format pdf
./generate.sh "公司名称" --format html
./generate.sh "公司名称" --format json

# 保存到指定文件
./generate.sh "公司名称" --output ./report.md

# 包含行业对标
./generate.sh "公司名称" --benchmark

# 邮件发送报告
./generate.sh "公司名称" --email user@example.com

# 生成评分卡
./generate.sh "公司名称" --scorecard
```

### 脚本参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `公司名称` | 要分析的企业名称 | 必填 |
| `--full` | 生成完整分析报告 | false |
| `--type` | 报告类型：all/risk/finance/news | all |
| `--format` | 输出格式：text/markdown/pdf/html/json | markdown |
| `--output` | 输出文件路径 | stdout |
| `--benchmark` | 包含行业对标分析 | false |
| `--email` | 邮件发送报告 | false |
| `--scorecard` | 生成评分卡 | false |
| `--template` | 自定义模板路径 | default |

## 风险评分算法

### 评分模型

```
综合风险评分 = Σ(维度权重 × 维度评分)

权重分配:
├── 运营风险: 25%
├── 财务风险: 30%
├── 法律风险: 20%
├── 市场风险: 15%
└── 声誉风险: 10%
```

### 评分等级

| 分数 | 等级 | 颜色 | 投资建议 |
|------|------|------|----------|
| 0-20 | 低风险 | 🟢 | 积极关注 |
| 21-40 | 较低 | 🟢 | 适度关注 |
| 41-60 | 中等 | 🟡 | 谨慎乐观 |
| 61-80 | 较高 | 🟠 | 风险警示 |
| 81-100 | 高风险 | 🔴 | 回避 |

## 缓存机制

- 报告默认缓存 12 小时
- 缓存目录：`~/.cache/wuwenzixun/report/`
- 强制重新生成：使用 `--force` 参数

## 常见问题

**Q: 报告生成失败？**
A: 检查网络连接，或使用 `--debug` 参数查看详细错误

**Q: 如何自定义报告模板？**
A: 使用 `--template /path/to/template.md` 指定模板

**Q: 能否批量生成报告？**
A: 支持传入企业列表文件：`--input companies.txt`
